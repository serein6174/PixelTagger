# 报告图片目录说明

本目录只存放 PixelTagger 实验报告采用的真实截图，推荐使用 PNG 格式。文件名采用两位序号、短横线和英文场景名，例如 `01-startup.png`、`02-open-single-image.png`；具体必需文件见上一级目录的 `SCREENSHOT_CHECKLIST.md`。

Markdown 从主报告引用图片时使用相对路径：

```markdown
![程序启动后的主界面](assets/01-startup.png)
```

截图前应关闭或裁去无关窗口，并对用户名、个人路径、仓库访问地址、Token、API Key、密码及私有服务器地址进行裁剪或不可逆遮挡。遮挡后仍应保留能证明功能与操作场景的界面上下文。

不得放置虚构截图、网络示意图或其他项目截图，不得提交私人密钥、凭据文件和包含敏感配置的图片。截图尚未取得时只更新清单状态，不要创建空白或伪造的 PNG 文件。
